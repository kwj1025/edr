# 프로젝트 공통 규칙

- **세션 초기화 시 기억 복원 (Context Restoration):**
  새로운 세션이 시작되어 사용자의 첫 명령을 받으면, 작업을 수행하기 전에 반드시 `chat_history.md` 파일의 최근 내역을 읽어(`read_file`) 이전에 진행했던 작업 내역과 프로젝트 문맥을 파악(기억)해야 합니다.

- **자동 대화 기록 (Chat Logging):** 
  사용자가 별도로 지시하지 않더라도, 사용자의 모든 요청 프롬프트와 에이전트의 답변(또는 작업 결과 요약)을 반드시 프로젝트 루트에 있는 `chat_history.md` 파일의 맨 끝에 기록(Append)해야 합니다.
  작성 시 가독성을 위해 반드시 `[YYYY-MM-DD HH:MM:SS]` 형태의 시간(Timestamp)을 표기하고, **User:** 와 **Agent:** 의 내용을 각각 줄을 바꾸어 작성하며, 각 대화 세트 사이에는 빈 줄과 구분선(`---`)을 넣어 겹치지 않게 확실히 분리하세요.
  작업을 완료하기 전에 잊지 말고 `chat_history.md` 파일을 업데이트 하세요.
